# 流光 Glance 项目文档

## 1. 项目简介

流光 Glance 是一个以“像素风桌宠 + 任务整理助手”为核心形态的桌面应用。项目将截图识别、活动流沉淀、任务管理、语音交互与 MQTT 硬件联动结合在一起，目标是把零散输入快速转成可执行待办。

当前推荐以桌面界面模式运行（`run_floating_app.py`），核心使用场景包括：

- 截图内容自动识别并提炼为任务
- 前台窗口变化持续采集，形成工作上下文时间轴
- 低频屏幕快照沉淀，支持回看与二次识别
- 结合活动流生成候选任务与今日日程摘要
- 任务通过 MQTT 同步到 ESP32 显示端
- 支持语音输入与语音播报

## 2. 技术栈

- 语言与运行时：Python 3.10+
- UI：Tkinter
- 图像能力：Pillow（截图、缩略图）
- 本地存储：SQLite
- AI 能力：OpenAI 兼容 SDK（默认接入阶跃星辰）
- 通信：paho-mqtt
- 其他：python-dotenv、PyYAML、pynput、pyperclip、requests

## 3. 核心架构

项目采用“单机桌面应用 + 本地数据库 + 可选硬件外设”的轻量架构，关键模块如下：

- `floating_app.py`：桌宠主界面与交互编排中心
- `run_floating_app.py`：桌面端入口脚本（推荐入口）
- `main.py`：后台服务入口（非桌面主入口）
- `task_manager.py`：任务的增删改查、优先级与截止时间管理
- `activity_manager.py`：活动流记录（截图/语音/系统/窗口采集）
- `foreground_window_watcher.py`：Windows 前台窗口变化采集
- `timeline_snapshot_manager.py`：时间轴快照抓取、缩略图生成、旧文件清理
- `llm_parser.py`：截图任务提取（视觉解析 + OCR 回退）
- `task_suggestion_engine.py`：活动流转候选任务
- `schedule_engine.py`：任务与活动融合生成日程摘要
- `voice_assistant.py`：语音识别、本地动作识别、模型回复
- `tts_stepfun.py`：TTS 语音播报
- `mqtt_client.py`：任务列表同步到硬件端

## 4. 关键业务流程

### 4.1 截图识别到任务

1. 用户触发截图（手动或快捷键/剪贴板）。
2. `llm_parser.py` 解析图片并返回结构化任务列表。
3. `task_manager.py` 写入 SQLite。
4. `mqtt_client.py` 将待办同步到 MQTT 主题。
5. 可选通过 `tts_stepfun.py` 进行语音提醒。

### 4.2 持续上下文采集

1. `foreground_window_watcher.py` 轮询当前前台窗口。
2. 窗口变化或满足重复发出间隔时写入活动流。
3. `timeline_snapshot_manager.py` 按策略抓取快照并生成缩略图。
4. UI 展示窗口时间轴与快照，并支持二次识别任务。

### 4.3 活动流整理为计划

1. `activity_manager.py` 提供近期活动记录。
2. `task_suggestion_engine.py` 生成候选任务（可一键采纳）。
3. `schedule_engine.py` 根据任务与活动输出“今日日程摘要”。

## 5. 数据与配置

### 5.1 本地数据

- 任务与活动默认存储于 `glance_tasks.db`
- 临时截图目录：`temp_screenshots/`
- 时间轴快照目录：`timeline_snapshots/`（运行时生成）

### 5.2 配置文件

- 主配置：`config.yaml`
- 环境变量模板：`.env.example`
- 建议将敏感信息放入 `.env`，避免直接写入配置文件

常用环境变量：

- `GLANCE_LLM_API_KEY`
- `GLANCE_TTS_API_KEY`
- `GLANCE_MQTT_BROKER`
- `GLANCE_MQTT_PORT`

## 6. 运行方式

### 6.1 推荐启动（统一脚本）

Windows：

```powershell
./start_dev.ps1
```

Linux/macOS：

```bash
./start_dev.sh
```

脚本会自动创建 `venv`、安装依赖并启动桌面端。

### 6.2 手动运行

```bash
python run_floating_app.py
```

## 7. 测试与质量

项目已在根目录 `test/` 下提供核心模块测试，执行方式：

```bash
python -m unittest discover -s test
```

当前测试覆盖重点：

- 任务管理
- 活动流管理
- 前台窗口采集
- 日程整理
- 候选任务整理
- 时间轴快照管理
- 语音助手动作识别

## 8. 平台与依赖说明

- 前台窗口采集与本地语音识别实现依赖 Windows 能力（Windows 优先）。
- MQTT 与模型服务可独立开关，不可用时不影响基础本地任务管理。
- 若 AI 或网络服务不可达，截图任务提取与 TTS 会降级或失败，但应用可继续运行。
